# Domain Docs

This repo uses a single product context.

## Before exploring, read these

- `CONTEXT.md` at the repo root for product language.
- `docs/adr/` for architectural decisions that constrain implementation.

## Use the glossary's vocabulary

When output names a domain concept, use the terms defined in `CONTEXT.md`. Avoid synonyms that the glossary explicitly rejects.

## Flag ADR conflicts

If work contradicts an ADR, surface the contradiction explicitly rather than silently overriding it.
